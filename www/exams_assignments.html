Please follow the official instructions for completing the HW 0. This page provides motivation, some context and help for completing the assignment.

Description:

Experience using database technology and systems is valuable. There are many technologies that we will use during the course. Installing and configuring the diverse software on personal devices can be difficult. The IT world is rapidly moving to cloud based services, including Database-as-a-Service (DBaaS).

We will primarily use Amazon Web Services (AWS). Please signup with AWS at the "free tier level." It is best to create a new AWS account for this class if you already have an AWS account. The signup process requires a credit card. We will avoid using services that create charges, and you also get a credit against services you use that incur charges. If you do not have a credit card, you can signup for an AWS Educate (Starter) Account. This should not require a credit card.

We will use the discussion thread on Piazza for HW0 to answer questions and resolve issues you encounter. Please post comments and questions. Also, please read the discussion thread before posting new questions or issues.

Administrators, analysts, developers and data scientists interact with databases using various tools. This semester, one tool we will use is JetBrains DataGrip. JetBrains provides free student licenses to all for all of their products. Please register, set up an account and get the license key. You can then download, activate and start DataGrip. You should download the professional edition.

Students who want to follow the programming track MUST also download and activate PyCharm. You can use the same license you received for DataGrip. Please download the professional edition. We strong recommend PyCharm for ll students.

The final setup step is installing Anaconda. Follow the instructions for "installing just for me." We recommend that you install a new instance in a new location if you already have a copy of Anaconda, but you can use an existing 3.7 or 3.8 installation.

We will provide a video tutorial for accomplishing some of the tasks for HW0. The HW0 tasks are:

  1. Signup for AWS, install DataGrip, install PyCharm and install Anaconda.
  2. Create a MySQL instances on the AWS Relational Data Service. Please follow the video tutorial.
  3. Register for a JetBrains account and get a license key. You will have an account and login information to activate your software. Download DataGrip, install and open to test your installation.
  4. Set up you connection between DataGrip and you Amazon RDS instance. There is a video tutorial that walks you through the steps.
  5. Anaconda:
    1. Download and install Anaconda. Follow the installation installations. Please make sure you follow the instructions for installing "just to me."
    2. There is a Jupyter Notebook that we produced to test set. Follow the instructions in the Jupyter Notebook to test your connection.

Homework 1: Introduction to Relational Databases

Please follow the official instructions for completing either the HW 1 - Programming or HW 1 - Non-Programming This page provides motivation, some context and help for completing the assignment.

Programming Description:

The first part of this homework will be more theoretical and will focus on different aspects of relational database systems. It was designed for students to better understand building blocks, namely different types of keys, relationships, and definitions. This section will also ask you to model a database using Lucid Chart . You will also become familiar with relational algebra, which is a theoretical declarative query language. These concepts are integral and important to understand before diving into the code and applications.

The next section of the homework is an application based where you will first "clean up" the lahman2019 database that you loaded in HW0. Data cleaning is an integral part of DBMS and is something that you will need to know for industry. In the homework we ask you to add primary and foreign keys, change column types, take care of null fields and add appropriate columns. You will need to do this for both the People and Batting Tables. All of this can be done within the DataGrips editor, where you can copy and paste the SQL Script generated from the Modify Table pop-up. For each change you make to the tables, we want to you explain why you are making the change and why it is important. Being a good data engineer does not just consist of knowing SQL and being able to work with different applications, but higher thinking and being able to explain every design choice you made. We then ask you some SQL queries, which should be fairly simplistic. You might want to check out documentation on aggregate functions.>

The final section of this homework is asking for you to implement basic query functions to manipulate CSV files. You will build on this code later in the semester, but for now it is important that not only does each function perform according to its definition but that you add integrity constraints. Try manipulating tables in SQL with primary keys, and figure out what you are allowed to do and what throws errors - your methods are simpler but should perform in the same manner.

When completed, please PDF your .ipnyb and upload it to GradeScope, then upload your .zip file containing your code, and all other items in the subsequent assignment section.
